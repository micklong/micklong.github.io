hexo被百度收录方法

  • hexo
  • 技术

前段时间，刚使用hexo在github上搭建了自己的博客，很顺利，网上的各种教程也很多。但是搭建好了之后，我相信大家都会遇到一个问题，自己的网站没办法被百度收录。当你使用百度站长去诊断测试的时候，网页的链接没有办法被百度蜘蛛爬到，所以网站没有被收录。我使用的是yilia主题，废话不多说，我把这个过程记录下来，希望后面对大家有一些帮助。需要用到以下几个步骤：

1.根目录下文件_config.yml ,添加一行配置,例如我的文件位置在 /_config.yml 。gitcafe和github的项目名最好是一样的，这样方便记忆。

github的服务器在国外，百度没有办法抓取到。所以得把项目备份到另外一个网站，也就是gitcafe上。也就是说，你在上传代码的时候，需要把代码同时传到github和gitcafe上去。那么有人问，这样会不会很麻烦，其实只需要一个简单的配置就能搞定了。

2.接下来就是域名部分，把gitcafe的解析线路设成默认的，github设成国外的就好了。

gitcafe是可以被百度爬虫收录的，github不可以。需要在购买域名的服务商上域名解析那做解析就好了，具体大家可以看下图。(ps:之前有人反馈说不带www的域名不能访问，是因为gitcafe没有提供A直接指向了ip地址，所以直接用cname指向gitcafe的项目域名就好了，特地更新了自己在万网上的域名解析截图)

3. 生成sitemap.xml。

在提交的时候，需要验证主站，百度站长里面也说明得很清楚，我就不细说了。在第二点的域名解析图片中，也加上了网站验证的截图。接下来就是要配置一下怎么生成sitemap,根目录下文件_config.yml中在最后添加以下几行命令。然后在根目录下执行命令 npm install hexo-generator-sitemap，之后重启博客，sitemap会发现已经生成了，然后把sitemap.xml放在\source的目录下，和CNAME同一层目录，这里面也可以放一下robots.txt。可以把sitemap提交到搜索引擎的站长平台来增加收录。如我的网站的目录为http://www.pmbuf.com/sitemap.xml ，如果是要在本地查看，就改成localhost:4000/sitemap.xml

当第2个步骤完成时，在百度站长里做抓取诊断，这时候应该是能被抓取到的，否则会抓取失败。在站长左侧的栏目中，有一个叫链接提交的功能，现在里面的sitemap,只需要把前面的sitemap地址http://www.pmbuf.com/sitemap.xml 填写进去就好了，不需要把里面的各个页面的链接单独拿出来，拿出去提交也是没有用的。如下图：

这个需要几天或者一个星期左右才能被收录，安心等待就好了，打完收工啦。
